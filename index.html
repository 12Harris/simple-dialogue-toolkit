Simple Dialogue Toolkit

A visual, node-based editor for branching dialogue trees, plus a small runtime that plays those dialogues back in-game. Build a graph by clicking around a canvas, save it as plain JSON, and drop that JSON into any Godot project that has the runtime autoloaded.

Get Started → Editor Guide

What's in the box

Visual graph editor

Right-click to add nodes, drag to wire connections, double-click any label to edit it in place. Everything autosaves to a plain JSON file you control.

Conditional choices

Attach an if expression to any player choice — it only appears as an option when the condition passes at runtime.

Auto-routed branches

Conditions nodes evaluate a list of if branches in order and route automatically, falling back to a trailing else case.

Blackboard variables

Conditions read from a simple key/value blackboard (DialogueManager.set_blackboard_variable()) that your game code controls — reputation, flags, stats, whatever you need.

Drop-in runtime

One autoload (DialogueManager) and a ready-made dialogue box UI — wire up its signals and you're playing dialogue in your own game.

Two ways to get it

PackageWhat it's for
Standalone app (Windows) Download, run, and try the editor plus a live playable demo — no Godot install required. Good for a quick look, or for authoring dialogue JSON to hand off to a consumer you write yourself in another engine.
Source package The dialogue/, ui/, utils/, assets/, data/, and scenes/ folders (minus the standalone-only launcher.tscn) as a plain download, meant to be dropped straight into your own Godot project. Gets you the editor and the runtime playback system as working, pluggable pieces.

Both packages share the exact same scenes and scripts — the standalone build just adds a launcher scene with a menu to pick between the editor and the demo.

How a dialogue file is shaped

Every dialogue is one JSON file: an object keyed by node id, where each node is exactly one of a plain line, a player choice list, or an auto-routed condition list. See Dialogue Format for the full schema.
