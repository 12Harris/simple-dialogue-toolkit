Editor Guide

Using the Editor

Everything is driven from a single canvas: right-click to add, drag to wire, double-click to edit.

Adding nodes

Right-click empty canvas space to open the Add Node menu, with three node types:

Node typeHasBehavior
Dialogue next A plain line that flows straight to the next node (or ends the dialogue if next is empty).
Choices choices Presents the player with clickable options; starts with 2 choices.
Conditions conditions Auto-routes based on blackboard state; starts with 1 if branch plus a trailing else.

Right-clicking anywhere else on a panel (not empty canvas) does nothing — no stray context menus while you're working inside a node.

Editing content

Double-click any label — the node's title, speaker, body text, or a choice/condition line — to edit it in place. It becomes a text field; press Enter or click elsewhere to save.

Edits auto-submit

If you double-click a different label, click the node itself, or click anywhere else (including empty canvas), whatever edit is currently open gets submitted first. You never lose a half-typed edit by clicking away.

The dialogue body text also supports a TW shortcut: end a line with TW and it's rendered as [tw] ("typewriter") in the saved file — a hint the runtime's text box uses to animate that line letter-by-letter instead of showing it instantly. This shortcut only applies to the body text field, not to choice/condition/title/speaker labels, so writing "TW" at the end of a choice label is left alone.

Choices and conditions rows

Both node types have a "+" button in the panel's bottom-right corner that adds a new row directly — no menu, no confirmation. Left-click it once, the row appears.

  • Both cap out at 6 rows. Once you hit the cap, the "+" button disables itself (with a tooltip explaining why) until you delete a row.
  • Right-click a choice row, or a condition row's if line (not the trailing else row), to open a small "Delete" popup for just that row.
  • A Choices node always keeps its first 2 choices; a Conditions node always keeps its first if (plus the else row, which is never deletable at all). Those rows don't get a delete option.

Writing conditions on a row

Double-clicking a row to edit it (see "Editing content" above) works differently for choice rows than for condition rows, since a choice needs both a condition and player-facing label, while a condition row is just the expression.

Choice rows

Double-clicking a choice opens a single text field containing the condition and the label together, separated by a pipe (|):

charisma >= 8 | Sweet-talk the guard.
  • Everything before the | is the if expression; everything after it is the choice's label.
  • Leave out the | entirely (or leave the part before it blank) to make the choice unconditional — typing just Turn back for now. with no pipe saves it as a plain choice with no if at all.
  • Adding a | to a previously-plain choice is what turns it conditional; removing the | from a conditional choice demotes it back to plain and drops its if.

Condition rows

Double-clicking a condition node's if row (not the trailing else row, which has nothing to edit) opens a text field with just the raw expression — no pipe, no label:

reputation >= 5

Validation

Whatever you type as the condition part is checked against the same <var> or <var> <op> <int> shape described in Dialogue Format. If it doesn't match, the edit is rejected and the row keeps its previous text — so a typo won't silently corrupt the row, but it also won't tell you what's wrong beyond declining to save.

The editor accepts == and != too

The editor's own validation is more permissive than the runtime: it accepts == and != as syntactically valid operators alongside <, <=, >, >=, so typing charisma == 8 saves without complaint. At runtime, though, both == and != are evaluated identically as "not equal" (see the == quirk on Dialogue Format) — so the editor won't warn you that an == condition doesn't mean what it looks like it means.

Wiring connections

  • Drag a node by its body to reposition it on the canvas.
  • Drag from a connector dot — the small squares on a node's left/right edge — to wire it to another node's next, a specific choice, or a specific condition branch.

Selecting and deleting

  • Left-click a node to select it.
  • Delete removes the selected node.

Navigation

  • Scroll to zoom.
  • Drag the background to pan.
  • R resets the view.

Saving

  • Ctrl + S saves.
  • The File menu has New / Load / Save / Save As.
  • The Load/Save dialog has a Recent panel listing places you've actually saved or loaded to (no manually-curated favorites list) — click an entry to jump straight there.
