Getting Started

Getting Started

Pick a package, open a scene, and you're editing dialogue in a couple of minutes.

Option A — Standalone app (Windows)

The quickest way to try the editor. No Godot install needed.

  1. Download and run the standalone build.
  2. From the launcher, choose Editor to open the graph editor, or Play Demo to try the runtime dialogue box.
  3. The editor loads a bundled example graph (data/castle_dialogue.json) by default — use File → Load Graph to open a different file, or File → New Graph to start from scratch.

This package is also a good fit if you're authoring dialogue JSON to hand off to a runtime you're writing yourself in another engine — the file format is plain JSON with no Godot-specific structure. See Dialogue Format.

Option B — Source package (drop into your own project)

Gets you the editor and the runtime playback system as pluggable pieces inside your own Godot project.

  1. Copy the dialogue/, ui/, utils/, assets/, and data/ folders into your project (merge assets/ with your own if you already have one — watch for filename collisions).
  2. Register the autoload — this is the one thing that doesn't travel with a folder copy, since it's stored in project.godot: go to Project → Project Settings → Autoload, add dialogue/DialogueManager.gd, and name it DialogueManager.
  3. To use the editor, open scenes/dialogue graph example scene.tscn and press Play (▶) on that scene specifically ("Run Current Scene", not your project's main scene). It loads data/castle_dialogue.json by default.
  4. To play a dialogue in your own game, see Integration for the two-line startup call and the signals you'll want to connect.
Same files, same behavior

Both packages share the exact same scenes and scripts. The only difference is the standalone build adds a launcher scene as an entry point with a menu to pick between the editor and the demo.

Project layout

scenes/    launcher.tscn (entry point), the editor scene, the demo scene
dialogue/  dialogue editor + runtime playback logic (DialogueManager, dialogue tree/graph)
ui/        UI widgets (dialogue box, text box, launcher buttons)
utils/     small generic utilities
assets/    fonts, icons, textures
data/      bundled example dialogue JSON
tools/     verify_reachability.py — standalone path-coverage checker
