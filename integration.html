Integration

Integrating into your own project

The source package is meant to be copied straight into an existing Godot project.

It intentionally leaves out project.godot, scenes/launcher.tscn, and its "Play Demo" button — you already have your own project settings and your own main scene.

1. Copy the folders in

Copy dialogue/, ui/, utils/, assets/, and data/ into your project. Merge assets/ with your own if you already have one, and watch for filename collisions.

2. Register the autoload

This is the one thing that doesn't travel with a folder copy, since it's stored in project.godot:

  1. Go to Project → Project Settings → Autoload.
  2. Add dialogue/DialogueManager.gd.
  3. Name it DialogueManager.

3. Using the editor in your project

Open scenes/dialogue graph example scene.tscn and press Play (▶) on that scene specifically — Godot's "Run Current Scene", not your project's main scene. It loads data/castle_dialogue.json by default; use File → Load Graph to point it at your own file, or File → New Graph to start from scratch. The "◀ Menu" button won't appear since there's no launcher in your project — that's expected.

4. Playing a dialogue in your own game

DialogueManager.load_dialogue("res://path/to/your_dialogue.json")
DialogueManager.start_dialogue()

Then connect to its signals to drive your own UI, or just instance scenes/main.tscn's DialogueBox node as a ready-made one. See dialogue/dialogue.gd for the minimal example the demo scene uses.

Signals

SignalFires when
dialogue_started start_dialogue() is called.
line_delivered(speaker, text) A plain line, or a choices/conditions node's own text, is ready to display.
choices_presented(choices) A choices node is reached — only choices with no if, or whose if currently passes, are included as clickable options.
check_conditions(conditions) A conditions node is reached — fires once that node's own displaytime wait has elapsed (see the note below), independent of whether the line itself has finished displaying. Evaluate branches in order and advance to the first passing one, or the fallback.
dialogue_paused_evt queue_pause_dialogue() was called and the dialogue reached a safe point to actually pause at.
dialogue_ended The dialogue reaches a node with next: null (or an equivalent dead end).
Conditions nodes have no Continue button

Since a Conditions node auto-routes based on blackboard state rather than a player choice, the bundled DialogueBox doesn't show a Continue button for one — instead it shows a small countdown label, driven by the same DT n / [displaytime:n] tag parsed from the node's own text (see Editor Guide). If a Conditions node's text has no displaytime tag, there's no visible wait — it advances immediately.

5. Blackboard variables

Variables used by if conditions (like "charisma >= 8") are read automatically from your dialogue file's condition strings and default to null. Set real values before starting the dialogue:

DialogueManager.set_blackboard_variable("charisma", 8)
DialogueManager.set_blackboard_variable("has_permit", true)
DialogueManager.start_dialogue()

See Dialogue Format for exactly how condition strings are parsed.
